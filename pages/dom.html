JavaScript and The DOM

Before we begin, what is the DOM?

The DOM, or Document Object Model, is the structure of an HTML page. It is the data representation of the objects that make up the DOM itself. With the DOM, programming languages like CSS and JavaScript can interact with web pages to create dynamic content, which improves user interactivity with the website, making the website more user friendly, and giving the user more reason to stay.

DOM Elements

The DOM always starts with the head and body tags. The head tag is used for data that is not displayed on the website itself. This includes any imports, stylesheets, and metadata. Metadata is data about data, such as a description of the webpage, the author, and keywords for search queries. This is also where the title tag goes, which is what will display on the search engine tab. The body tag contains everything that the website will display, including links to other pages, content, buttons, etc. This is where you put most of your HTML, as well as your CSS and JavaScript.

When interacting with the DOM, we use CSS and JavaScript, which are the only languages (other than HTML) that can run in the browser without using plugins. To access the DOM, we use JavaScript methods. To retrieve an element from the DOM, we use the document.querySelector() and document.getElementById() methods. The former takes any number of CSS selector strings to match to, and returns the first matching element. The latter takes the id attribute of the element you want to fetch. To use these methods however, we need a JavaScript function. If you try to write vanilla JavaScript in a plain HTML document, you will get an error, so how do we get around this? The answer is to use the script tag. By wrapping our vanilla JavaScript code in the script tag, we can write a function in vanilla JavaScript to interact with the DOM using the above methods. If you open your inspection tool, you can see both functions between script tags. By clicking each button, the text inside the box changes, and each button is behaving in a different way based on the script. Try it out, and then read on for an in depth explanation of what's going on.

Watch What Happens!

Inside the script tags, you will see the vanilla JavaScript code that is modifying the text tag in real time. The document.querySelector() method finds the first element that matches the CSS selector, which in this case is looking for a class name. It then uses the textContent property to change what is displayed on the screen inside the text tag. The document.getElementById() method is similar, but is retrieving elements only by id. Your HTML code should never have a duplicate id field, so make sure you are giving each element its own id attribute. On this page, we only covered retrieving elements. However, we can do many other kinds of operations on the DOM, including creating elements, appending new elements to existing elements, changing the style of an element, etc. There is so much to vanilla JavaScript that can be explored, and I encourage you to do a deep dive into the Mozilla developer documentation to learn as much as you can!
